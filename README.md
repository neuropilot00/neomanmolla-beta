# 너만 몰라

혼자서도, 4~6명이서도 짧게 즐기는 모바일용 소셜 추리 게임 베타입니다.

한 명만 다른 질문을 받고, 모두의 답변을 공개한 뒤 누가 어색한지 찾아냅니다.

## 실행

프론트는 정적 파일로도 열 수 있지만, 실제 베타 테스트는 Railway 서버 URL에서 진행합니다.

Railway 서버 베타는 아래처럼 실행합니다.

```bash
npm start
```

프론트는 기본적으로 Railway 운영 API를 사용하며, 일반 유저에게 API URL 입력칸을 노출하지 않습니다.

Railway의 `neomanmolla-beta` 서비스에는 아래 변수를 설정합니다.

```txt
DATABASE_URL=${{Postgres.DATABASE_URL}}
```

Postgres가 같은 Railway 프로젝트 안에 있을 때는 공개 TCP 프록시 주소보다 내부 `railway.internal` 주소를 쓰는 `DATABASE_URL` 참조가 맞습니다.

## 베타 테스트 방법

1. `빠른 참가`로 공개 대기방에 들어가거나 `방 만들기`에서 질문팩과 테마를 선택합니다.
2. 서버 방이 생성된 뒤 대기실에서 `초대 링크 복사`로 플레이할 사람에게 링크를 보냅니다.
3. 링크를 받은 유저는 닉네임만 입력하고 같은 방에 참가합니다.
4. 플레이 후 결과 화면에서 각자가 받은 질문과 답변을 확인합니다.

일본어 테스트는 첫 화면에서 `日本語`를 선택하거나 링크에 `lang=ja`를 붙이면 됩니다. 한일 혼합 테스트는 방 준비 화면에서 플레이어별 언어를 선택하면 같은 라운드가 각자 언어로 표시됩니다.

K-pop 그룹 데이터는 내부 테스트용 큐레이션입니다. 정식 공개 전에는 권리/표기/업데이트 정책을 별도 확정해야 합니다.

현재 버전은 Railway API와 Postgres로 방 상태를 저장하고, 프론트가 주기적으로 방 상태를 갱신하는 1차 운영형 베타입니다.

## Railway API

현재 API는 외부 의존성 없이 Node 기본 `http` 모듈로 동작합니다. 같은 저장소를 Railway에 연결하고 `npm start`를 실행하면 됩니다.

주요 엔드포인트:

- `GET /health`
- `GET /api/content`
- `POST /api/rooms`
- `GET /api/rooms/:code`
- `POST /api/rooms/:code/join`
- `POST /api/rooms/:code/start`
- `POST /api/rooms/:code/answers`
- `POST /api/rooms/:code/votes`
- `POST /api/match/quick`

서버는 `DATABASE_URL`이 있으면 Postgres에 `rooms` 테이블을 자동 생성하고 방 상태를 저장합니다. 아직 Socket.IO 실시간 푸시는 붙이지 않았고, 클라이언트가 API 폴링으로 상태를 동기화합니다.

## 확인할 점

- 첫 화면에서 룰을 10초 안에 이해할 수 있어야 합니다.
- 결과 화면에서 공통 질문과 가짜 질문의 차이가 바로 보여야 합니다.
- 최소 3판 이상 반복 플레이 의향을 확인해야 합니다.
- 초대 링크를 실제 친구에게 보낼 만큼 민망하지 않아야 합니다.
